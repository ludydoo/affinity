Class: Attribute

Attribute

A Relation's Header is composed of Attributes.

An Attribute is simply an ordered Set {name : String, type : Type}

The Type of an Attribute is a class that is mainly used to compare the values of the Relation's tuples for this attibute.

For example, when running an RUnion on a Relation, the Tuples will be tested for equality so that the Relation does not include duplicates (as the mathematical definition of a Set tells us). To do this, we compare two Tuple by comparing the value of their Attributes. If for these two Tuples, all their Attribute values are equal, the two Tuples are said to be equal.

To test the equality of primitive Attributes (String, Boolean, Integer...), this is easy. We test the equality of two Attributes with the equality operator "==".

Though, when it comes to Object Attributes, it is not possible to test the equality with the equality operator "==". We need a function that will test the two objects for equality and that will return true/false.

To resume, the Type of an Attribute must implement the methods used to compare the corresponding Tuple Attribute values.

For TNumeric Types, the Type will also implement other comparison operators such as LargerThan (.lt()) ...

For TDate Types, the Type will also implement other operators such as DayOfWeek, ...

Example
var fn1 = new affinity.Attribute({name : 'FirstName', type : affinity.String});

var fn2 = {name : 'FirstName' , type : affinity.String}

affinity.Attribute.coerce(fn1) === fn1 // true, the object was already an Attribute.
affinity.Attribute.coerce(fn2) === fn2 // false, the object had to be coerced

affinity.Attribute.coerce(fn2).equal(fn1) // true
